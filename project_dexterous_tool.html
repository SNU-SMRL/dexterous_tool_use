Language-Conditioned Dexterous Tool Use via GR00T N1.7

GR00T N1.7의 dexterous manipulation prior (EgoScale 20K시간 인간 영상 사전학습)를 contact-rich tool use로 확장하여, 기존 VLA가 다루지 못한 language-conditioned dexterous tool use를 Isaac Lab 시뮬레이션에서 구현한다.

3–4개월 RTX 5080 Isaac Lab GR00T N1.7 (3B)
1연구 배경 — 세 흐름의 미합류

세 독립적 연구 흐름이 존재하며, 그 교차점이 비어 있다.

Flow 1
VLA Models
π0/π0.5, GR00T N1.7, OpenVLA. 강력한 semantic understanding이나 tool use는 미검증. GR00T N1.7은 EgoScale로 22-DoF hand까지 확장했지만, 태스크는 rearrangement에 한정.
Flow 2
Dexterous RL
OpenAI Rubik's Cube, Visual Dexterity (Science Robotics 2023), DeXtreme (ICRA 2023), DexReMoE. Finger-level 제어는 성숙하나 시맨틱 이해 없음.
Flow 3
Dexterous Tool Use
SimToolReal (Stanford, 2026), Grasp-to-Act (UIUC, 2026), DexMachina (Columbia/NVIDIA, 2025). 도구 affordance 기반 조작하나 VLA 통합 없고, 인간 시연 비디오 필요.

3핵심 Gap 3가지
Gap A
Language → Tool Trajectory 자동 생성
SimToolReal은 goal pose가 주어지면 도구를 다루지만, 인간 비디오에서 추출해야 한다. 자연어 명령에서 자동 생성하는 시스템 부재.
Gap B
VLA의 Tool Use 미검증
GR00T N1.7이 dexterous manipulation을 수행하지만, rearrangement에 한정. Contact-rich tool use는 범위 밖.
Gap C
Grasp–Reorient–Use 통합
도구를 기능적으로 잡기 → 손 안 자세 조정 → 실제 사용. 이 full chain을 VLA로 엮은 연구 없음.
4시스템 설계

Core Question

GR00T N1.7의 dexterous manipulation prior를 tool use 태스크에 post-training하면 language-conditioned dexterous tool use가 가능한가?

GR00T N1.7을 선택한 이유 (vs π0.5)

  • EgoScale로 22-DoF dexterous hand 사전학습 완료 (π0.5는 dex hand 데이터 없음)
  • Embodiment adapter 내장 → XL-VLA latent space 별도 구현 불필요
  • Isaac Lab과 같은 NVIDIA 생태계, 오픈 모델 + 상업 라이선스
  • Inference 16GB+ VRAM (5080 가능), Fine-tuning 40GB+ 추천

GR00T N1.7에서도 Tool Use가 안 되는 이유

  • 학습 데이터가 rearrangement (물체 옮기기) 위주
  • Contact-rich interaction (충격, 토크, 연속 저항) 학습 데이터 부재
  • Tool-environment interaction semantics가 인코딩되지 않음

데이터 파이프라인

Isaac Lab에서 RL expert 또는 pretrained/scripted policy를 실행하여 매 스텝 (RGB, language, joint, tool pose)를 녹화. 성공 에피소드 필터링 후 language annotation 자동 생성하여 5K~10K trajectory 확보.
